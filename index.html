Using Git & GitHub

Git is powerful and popular version control software that is used for making collaboration on projects easy, as well as keeping detailed records of how each individual file in a project has changed over time. GitHub is an online service that allows you to host your Git repositories remotely. GitHub offers both free and paid accounts which both allow you to host unlimited repositories on their site. The only differnce between free and paid accounts is that a paid account is required to keep a number of your repositories private and unable to be seen by those who haven't been added as collaborators. Creating A Repository

The very first thing you need to do to use Git (assuuming you already have it installed) is to create a repository. This can done via the command line using the git init command, or through a Git/GitHub application. In either case, all you have to do is navigate to the directory where your project is stored and initialize the repository, either with the command mentioned earlier, or via the "Create" button in your Git application.

Creating Branches

When you create a Git repository, by default, you'll make changes to what is known as the master branch. This is the branch from which you'll make deployments.
