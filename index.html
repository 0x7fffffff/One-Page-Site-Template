Using Git & GitHub

Git is powerful and popular version control software that is used for making collaboration on projects easy, as well as keeping detailed records of how each individual file in a project has changed over time.

GitHub is an online service that allows you to host your Git repositories remotely. GitHub offers both free and paid accounts which both allow you to host unlimited repositories on their site. The only differnce between free and paid accounts is that a paid account is required to keep a number of your repositories private and unable to be seen by those who haven't been added as collaborators. Below is a description of the basic workflow that you'll use when interacting with Git and GitHub.

GitHub Workflow

  1. Creating A Repository
  2. Creating Branches
  3. Making Commits
  4. Pull Requests & Merging Branches
  5. Creating Releases

Creating A Repository

The very first thing you need to do to use Git (assuuming you already have it installed) is to create a repository. This can done via the command line using the git init command, or through a Git/GitHub application. In either case, all you have to do is navigate to the directory where your project is stored and initialize the repository, either with the command mentioned earlier, or via the "Create" button in your Git application.

Creating Branches

When you create a Git repository, by default, you'll make changes to what is known as the master branch. This is the branch from which you'll make deployments, and will often make changes to directly. However, you may be planning a new feature for your project that will cause a lot of changes that you don't necessarily want to be deployed yet. A great way to do this is to create a new branch from your master branch. Doing this allows you to make changes alongside the main project, without actually modifiying the deployable branch. This can be done in the GitHib application by navigating to the "new branch" button and following the prompts.

Making Commits

After you add files to your project or make changes files already in your project, you need to to tell the repository that it's time to save a record of these changes. You do this by creating what is known as a commit.

To create a commit, the first thing you need to do is stage the files that have chnaged that you wish to commit. This is most easily accomplished in the GitHub application's commit menu, in which you can simply check off which files that you want to stage. After that, all you have to do is type in the message that you want to have associated with that commit so that it can be recognized in the future and hit the commit button. Doing so will save the commit locally within the repository, so if you want to push your changes up to the server (GitHub) you must then hit the sync button.

Hint: Git diffs show you the differences between your current commit and the last commit (red/green highlighting)

Pull Requests & Merging Branches

Now that you've made changes to your project and committed them, you're going to want to apply these changes. There are two common ways of doing this. The first to merge the branch you've created back into the master branch. This is usually useful when making changes to a repository hosted locally. Merging the branch into the master branch will bring control flow back and the repository's working head back to master. After doing so, you will no longer make commits to the branch you've created, but to the master branch instead.

The other common workflow for applying changes is to make a pull request to the repository. This is common when working on a remotely hosted project with multiple collaborators, or an open source project of some kind. In this case, you'll navigate the the repository's branches section on GitHub and select the branch you've been working on. From there, you'll be given the option to merge the pull request back into the repository's master branch.

This is very similar to the way you merge branches yourself but with one major difference. Instead of just being able to merge the branches, you have to request that it be done by the project's owners.

Creating Releases

Once you've made all your changes and your event tree is looking good, you may want to publish a specific events in the history of the project. (v1.0, v2.0, etc.) To do this, you create a release. Creating releases are easy. All you have to do is navigate to the releases section for your repository on GitHub and click the "Create New Release" button. From there, you'll see a menu that allows you to create a release of a specified version number and title. Users will be able to download your project as it was at this point by downloading this release.

Sublime Text 2 Training Plan

Effectively Using Sublime Text 2

Sublime Text 2 is an extremely flexible and powerful text editor that can be used for programming in any language.

Following this guide should give you a basic understanding of some of the features of Sublime Text 2 and help you to learn how to start using it proficiently. In this training exercise I’ll be explaining how to use some of the features of Sublime Text 2 that I find most useful when using it myself. This will include features like, the side navigator, multiple selections, the command palette, templates and the find an replace menus.

This guide will assume that you already have downloaded and installed Sublime Text 2 from http://www.sublimetext.com and are comfortable with basic programming concepts. Other than that no tools will be required for this guide.

This guide will use the learn by doing training technique coupled with brief verbal descriptions of each topic when the topic requires a deeper understand than can be quickly gained from viewing the program. Guidance will be applied as necessary if the student has any additional questions or concerns about Sublime Text 2. In other situations, the student will watch me show them how to use a particular feature.

Curriculum

  • The Side Navigator
    • 2 Minutes
    • Show/ explain the feature without user interaction.
    • Demonstrate how the side bar can be a useful tool for quickly jumping around between large sections of code by seeing the entire page in a zoomed out scrolling view.
  • Multiple Selection
    • 3 Minutes
    • Start by explaining the feature and showing it to the student. From there, allow student to try using the feature themselves so that they can see how powerful of a tool it is.
    • Demonstrate the usage of multiple selection in Sublime Text 2. Show different ways to add multiple cursors to the screen and how this can increase productivity.
  • The Command Palette
    • 4 Minutes
    • Start by explaining what the command palette is and show a few examples of what it can do to increase productivity. Then let the student try to install a few packages to show them how easy it is to interact with.
    • The Command Palette allows for the use of any command in Sublime Text 2 to be executed without remembering the keyboard shortcut for it. After activated, the command palette allows the user to fuzzy search the names of allow available commands. It also includes and extensive package manager to allow the installation of plugins for Sublime Text 2.
  • Templates
    • 3 Minutes
    • Explain and demonstrate what templates are and show how they can save you a lot of time while writing programs.
    • After demonstration is over, let the student try using templates for themselves using preexisting templates or let them define their own.
  • Find and Replace Menus
    • 3 Minutes
    • Show the different features of Sublime Text 2’s find and replace menus and how these can differ from other editors.
    • Sublime Text 2 offers advanced find and replace menus that allow for more than just simple find and replace. You can match searches against current project status, as well as past commits (if version control is enabled). You can also find/replace matches using inline Regex.

Contingency Plan

  • Problem - Unable to download or install Sublime Text 2 on student’s computer
    • Solution - Make demonstration on my own computer.
  • Problem - Side menu is not enabled by default.
    • Solution - Navigate to application’s settings.json file and modify side bar state.
  • Problem - Package manager can’t be used due to lack of Internet connectivity.
    • Solution - Only demonstrate offline features of the command palette and describe what can be done with the package manager.
  • Problem - Corrupt template files generated during exercise.
    • Solution - Sublime Text 2 template rebuild, or if necessary manually make corrections myself.
  • Problem - Lack of ability to remember useful Regular Expressions.
    • Solution - Look up examples online and apply them to the find/replace section.
